# Contributing

Read `AGENTS.md`, `docs/COMPLIANCE_BOUNDARIES.md`, `docs/DEVELOPMENT_RULES.md`, and `docs/RISK_REGISTER.md` before making changes.

Use Conventional Commits, for example `chore: initialize Better Star Rail repository`.

The repository's current primary license is Apache License 2.0. BetterGI's GPL-3.0 status is recorded only as third-party reference information; its code must not be copied unless a future project-owner decision changes the licensing strategy. Repositories without a license may be observed for public behavior only; do not copy their code, assets, wording, configuration, coordinates, or thresholds.

Do not submit credentials, tokens, cookies, private keys, local machine paths, game assets, copied source code, screenshots from copyrighted materials, or unlicensed binaries.

Every PR must state whether it touches screen capture, input simulation, user data, networking, dependencies, or project scope. Red-line features such as injection, memory access, packet manipulation, anti-cheat bypass, hidden execution, credential collection, and automated payment/resource spending are not accepted.

Every PR must list all third-party code, documentation, assets, tools, and reference projects consulted. If none were used, state that the change is an independent clean-room implementation. A public repository is not automatically permission to copy its contents.
