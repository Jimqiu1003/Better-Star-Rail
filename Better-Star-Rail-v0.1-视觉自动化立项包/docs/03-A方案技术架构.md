# 03 A 方案技术架构

## 方案名称

Vision-Only UI Automation

## 技术原则

只从用户可见画面获取状态，只通过普通前台键鼠输入执行动作。

## 技术栈

- C# / .NET 10 LTS
- WPF + MVVM
- OpenCvSharp 或许可证兼容的 OpenCV 封装
- OCR 引擎通过独立适配层接入
- Windows Graphics Capture 或项目批准的截图方式
- xUnit
- GitHub Actions

引入任何库前必须审查许可证、二进制来源和供应链风险。

## 分层结构

```text
src/
  BetterStarRail.App/              # WPF UI、风险确认、任务控制
  BetterStarRail.Application/      # 用例、调度、状态机
  BetterStarRail.Domain/           # 页面、任务、动作、安全规则
  BetterStarRail.Capture/          # 窗口截图
  BetterStarRail.Vision/           # OCR、模板匹配、页面识别
  BetterStarRail.Input/            # 受控键鼠执行
  BetterStarRail.Automation/       # 流程、状态机、恢复
  BetterStarRail.Recorder/         # 录制与回放
  BetterStarRail.Infrastructure/   # 配置、日志、通知
tests/
```

## 核心数据流

```text
目标窗口
  -> 截图
  -> 图像预处理
  -> 页面识别
  -> 安全条件校验
  -> 状态机决策
  -> 输入动作
  -> 动作后截图验证
  -> 成功 / 重试 / 停止
```

不得出现“先点击再猜结果”的无限循环。

## 状态机要求

每个自动化任务必须定义：

- 起始页面；
- 可接受页面集合；
- 页面进入条件；
- 可执行动作；
- 动作前条件；
- 动作后验证；
- 超时；
- 最大重试；
- 可恢复状态；
- 不可恢复状态；
- 结束条件。

## 输入执行器限制

输入模块必须具备统一策略：

- 只有目标窗口位于前台且句柄与校准一致时才允许输入。
- 每个任务拥有动作白名单。
- 输入频率有上限。
- 用户真实输入优先；检测到用户干预时暂停。
- 紧急停止不可由任务代码屏蔽。
- 不提供随机轨迹、反检测或行为伪装 API。

## 截图与隐私

- 截图默认保存在本地。
- 失败截图可配置自动过期。
- 导出前显示预览并提醒可能包含 UID、昵称或聊天内容。
- 禁止静默上传。
- 测试仓库只使用合成界面或取得授权的素材。

## 插件策略

v0.1 不开放任意 DLL、Python、JavaScript 或 Lua 插件。

任务脚本只能使用项目定义的受限声明式动作：

```text
WaitForPage
RecognizeText
MatchTemplate
ClickTarget
PressAllowedKey
Delay
AssertState
Stop
```

脚本不得访问文件系统、网络、进程、注册表或执行任意命令。
