# Better Star Rail v0.1 — A 方案立项包

> 历史文档提示：许可证与复用规则以仓库根目录的 `docs/decisions/ADR-0004-approved-baseline-supersession.md` 为准。项目采用 Apache-2.0 clean-room 独立实现，不复制 BetterGI 或 Better-HSR-Currency-Wars 的源码、素材、模板、文案、坐标和阈值。

> 项目定位：面向 Windows 的《崩坏：星穹铁道》非官方视觉自动化工具。  
> 风险等级：高。第三方软件、模拟操作与无人值守功能可能违反游戏用户条款，并可能导致账号处罚、功能失效、软件被拦截或项目被要求停止。

## 1. 核心决策

v0.1 采用 **A 方案：屏幕视觉识别 + OCR + 前台键鼠模拟 + 有限状态机**。

项目明确接受类似 BetterGI 的技术风险，但不承诺：

- 不封号；
- 永久可用；
- 不被反作弊、安全软件或系统拦截；
- 不发生误操作；
- 与未来游戏版本兼容；
- 权利方认可或允许本项目。

免责声明不能免除开发者依法承担的责任，也不能改变游戏运营方依据用户条款采取措施的权利。

## 2. v0.1 必须完成

- 自动启动官方游戏或启动器。
- 自动消耗开拓力。
- 自动进入指定副本并使用游戏内自动战斗。
- 自动循环挑战、结算和领取奖励。
- 自动完成或领取委托。
- 自动领取邮件、每日训练与活动页面中的可领取奖励。
- OCR/模板识别后自动点击。
- 键鼠录制、回放和任务编排。
- 在用户明确启用后进行有限的无人值守运行。
- 运行日志、截图证据、异常恢复与紧急停止。

## 3. 绝对红线

即使用户主动要求，也不进入本项目：

- 读写游戏内存。
- DLL 注入、图形 Hook、内核驱动。
- 禁用、绕过、欺骗或探测反作弊。
- 隐藏进程、伪造签名、随机化行为以规避检测。
- 抓包复现私有协议或调用未公开接口。
- 收集账号、密码、Cookie、Token 或验证码。
- 自动充值、购买付费商品或消耗星琼。
- 多账号托管、代练、账号交易和远程云控。
- 修改或重新分发官方游戏文件。
- 宣称“安全”“零封号”“官方认可”。

## 4. 开源与代码来源

- 项目建议采用 `GPL-3.0-only`。
- BetterGI 可以作为架构和交互参考；复制其 GPL 代码时必须保留版权与许可证，并履行 GPLv3 的源码分发义务。
- 没有明确许可证的仓库不得复制代码、素材、OCR 模板、配置或文案。
- 官方游戏素材不得直接打包进发布文件；识别模板优先通过用户本机截图生成，或使用团队原创/明确授权的测试素材。

详细内容见 `docs/`。

# 02 需求规格 v0.1

## 用户与环境

- Windows 10/11 x64。
- 官方 PC 客户端。
- 首期仅支持项目明确列出的显示比例、分辨率和界面语言。
- 用户必须在首次启动时主动阅读并确认风险。
- 未确认风险时，仅允许进入设置和文档页面。

## P0-01 自动启动游戏

### 需求

- 支持用户选择 HoYoPlay 或官方游戏入口。
- 可配置延迟启动和任务计划。
- 检测启动器/游戏窗口是否出现。
- 遇到登录、验证码、用户协议、更新或异常弹窗时暂停，要求用户处理。

### 禁止

- 不保存账号、密码、Cookie、Token 或验证码。
- 不自动填写登录凭据。
- 不绕过启动器或游戏更新。

## P0-02 自动消耗开拓力

### 需求

- 用户选择允许执行的副本、队伍、次数和最低剩余开拓力。
- 根据界面识别当前开拓力。
- 自动进入副本、开始挑战、调用游戏内自动战斗、结算并继续。
- 达到次数、时间、开拓力下限或失败阈值后停止。

### 默认保护

- 禁止自动使用星琼补充开拓力。
- 禁止自动使用燃料、梦之泪滴等稀缺资源，除非未来作为独立高风险功能重新评审。
- 当 OCR 结果低于置信度阈值时停止。
- 任何按钮位置与预期不符时停止。

## P0-03 自动刷副本与战斗

### 需求

- 支持进入指定副本。
- 支持选择或验证队伍。
- 支持启动游戏内自动战斗。
- 识别胜利、失败、掉线、角色阵亡、体力不足和结算页面。
- 支持有限次数重试。

### v0.1 不做

- 不实现实时战斗 AI。
- 不依据内存数据选择技能。
- 不进行高难模式、排名模式或对抗玩法自动化。
- 不自动更换装备、分解遗器或执行不可逆养成操作。

## P0-04 委托处理

### 需求

- 进入委托页面。
- 领取已完成委托。
- 根据用户白名单重新派遣。
- 识别派遣时长与角色冲突。

### 默认保护

- 不自动更改用户未授权的委托槽位。
- 识别不确定时跳过该槽位。
- 保存操作前后截图到本地调试目录，可由用户关闭。

## P0-05 邮件、每日训练和活动奖励

### 需求

- 自动打开相关页面。
- 仅点击明确识别为“可领取”的奖励。
- 识别奖励领取完成状态。
- 对活动页面使用版本化流程，不使用通用“到处点红点”。

### 禁止

- 不点击付费、购买、兑换、跳转网页、账号绑定按钮。
- 不自动删除邮件。
- 不自动兑换第三方兑换码。
- 不操作包含星琼消费确认的页面。

## P0-06 OCR 与模板识别后点击

### 需求

- 截图源抽象化。
- ROI 区域管理。
- 模板匹配、OCR、颜色/几何规则可组合。
- 每次动作必须由“页面状态 + 目标控件 + 安全条件”共同确认。
- 所有坐标基于窗口客户区归一化，不写死屏幕绝对坐标。
- 支持置信度阈值、超时、重试和失败截图。

### 规则

禁止只因为 OCR 识别到一个词就立即点击；至少需要第二个页面或几何条件验证。

## P0-07 键鼠录制与回放

### 需求

- 用户明确按下开始/停止进行录制。
- 仅记录目标游戏窗口客户区内的鼠标与键盘事件。
- 回放前显示动作数量、预计时长和风险。
- 支持暂停、单步、速度限制、循环次数上限和全局停止。
- 录制文件使用可读、可审查的格式。

### 禁止

- 不记录密码输入框。
- 不记录系统全局键盘输入。
- 不后台静默录制。
- 不提供规避检测的随机延迟、随机轨迹或行为伪装选项。

## P0-08 无人值守调度

### 需求

- 必须由用户主动开启。
- 默认单次运行，循环调度需要额外确认。
- 设置最大运行时长、最大点击数、最大重试次数。
- 每个阶段都有看门狗和心跳。
- 检测到验证码、登录失效、更新、异常弹窗、非预期窗口或识别低置信度时停止。
- 结束后生成执行摘要。

### 默认限制

- 单次最大运行时长：60 分钟。
- 连续失败：3 次后停止。
- 单一页面重复动作：5 次后停止。
- 用户可自行调低，不得无上限运行。

## P0-09 安全控制

- 全局紧急停止快捷键。
- 鼠标移至屏幕角落可触发停止。
- 主界面常驻停止按钮。
- 失去目标窗口时停止输入。
- 窗口尺寸、DPI、语言或分辨率变化时停止并重新校准。
- 自动化执行时显示明显状态，不允许隐身运行。

# 05 绝对禁止项

## 反作弊与规避

禁止开发、接受贡献或讨论实现：

- 关闭、卸载、绕过或欺骗反作弊。
- 检测反作弊扫描时自动隐藏。
- 进程名、窗口名、签名或文件哈希伪装。
- 随机化轨迹、时间和行为以模拟真人或降低检测概率。
- 内核驱动、句柄提权、调试器规避。
- 远程下发反检测策略。

## 游戏进程和协议

禁止：

- ReadProcessMemory / WriteProcessMemory。
- DLL 注入、Remote Thread、图形 Hook。
- 扫描模块、偏移、实体或战斗数据。
- 抓包、解密或复现私有协议。
- 调用未公开 API。
- 修改客户端、配置、资源包或游戏文件。

## 账号与支付

禁止：

- 采集或存储账号密码。
- Cookie、Token、验证码和浏览器会话。
- 自动充值、购买付费商品。
- 自动消耗星琼。
- 账号托管、租赁、交易和代练。
- 多账号农场和云端批量控制。

## 高损失操作

v0.1 禁止：

- 遗器分解、销毁、强化。
- 角色、光锥和遗器养成。
- 抽卡。
- 商店购买。
- 好友删除、社交消息发送。
- 邮件删除。
- 修改账号或隐私设置。

## 发布和宣传

禁止：

- 宣称不会封号或绝对安全。
- 冒充官方或使用官方视觉身份。
- 引导用户关闭反作弊。
- 把安全软件警报统一描述为误报。
- 隐藏真实功能或使用无害名称规避审核。
