# Security Policy

Report security issues privately to the project maintainer until a public process is established. Do not publish exploit details, credentials, or user screenshots in public issues.

Critical issues include credential leakage, unsafe input dispatch, uploads without consent, copied unlicensed code, anti-cheat bypass logic, process injection, memory access, or any feature that can consume paid or high-value resources without explicit confirmation.

The project does not provide anti-cheat bypass, account-risk evasion, or guarantees that automation is safe under game terms of service.
