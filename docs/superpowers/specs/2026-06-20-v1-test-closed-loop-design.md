# V1 自建测试窗口安全闭环设计

## 目标与边界

V1 只面向标题和窗口类均由项目定义的自建测试窗口，完成“截图、页面识别、安全校验、输入、动作后验证、成功或安全停止”闭环。不查找、不启动、不读取《崩坏：星穹铁道》进程，也不包含游戏业务、无人值守、宏、脚本、注入、Hook、内存或规避检测能力。

## 架构

- `Core` 保存平台无关的窗口身份、客户区、归一化坐标、输入动作和急停契约。
- `Vision` 保存 BGRA 帧、ROI、五种自建页面的 `PageDetectionResult`，使用项目自制纯色锚点识别器；Golden Image 为文本 PPM，不含第三方或游戏素材。
- `Automation` 保存统一 `SafetyGuard` 和闭环状态机。每次输入前重新读取窗口和页面，检查句柄、前台、客户区、DPI、用户输入世代、置信度、白名单和急停。
- `Infrastructure.Windows` 通过 Win32 获取窗口/客户区状态，使用 GDI `BitBlt` 捕获客户区，通过定向窗口消息实现测试窗口点击和允许按键；所有句柄在 `finally` 中释放。
- `BetterStarRail.TestWindow` 是团队自建的原生 Win32 测试目标，提供 Welcome、Confirm、Progress、Complete、Blocked 五个页面，不含游戏素材。

## 数据流与故障安全

状态机只允许 `Welcome -> Confirm -> Complete` 两次页面跳转。每一步都有预期页、动作、预期后置页、超时、最大尝试次数和失败原因。Unknown、低置信度、窗口变化、失焦、用户输入、急停、取消或超时立即停止；失败帧和不含敏感数据的 JSON 诊断只写本地。

## 测试策略

单元测试覆盖归一化坐标、五页正样本、负样本、低置信度、Unknown、动作白名单、失焦、错误窗口、窗口/DPI 变化、用户输入、急停、取消和超时。内存测试窗口与 Golden Image 使用同一页面定义，执行 100 次完整闭环；原生适配器只做可控参数和资源路径测试，CI 不启动游戏或发送真实桌面输入。

## 规则冲突记录

用户级“每次修改复制到版本号子目录”与已接受的 ADR-0003、解决方案项目路径及“小型 feature 分支”要求冲突。本实现遵循更具体且可审计的仓库结构，使用 feature 分支、提交和应用版本号追踪变更，不复制整仓到 `v1.0/`。
